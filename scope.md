# Project Scope Statement

## Project Purpose
Writing the scope first helps me stay focused on what the page actually needs instead of adding things that weren't required. I want the page to introduce who I am, share some of my professional interests, and give a short overview of my goals. 

## Description
This project includes two parts: a scope statement written in Markdown and a single-page website written in HTML. The webpage will include my name, my professional interests, a short description of my goals, a list of hobbies or interests, and a link to my GitHub profile. The main focus is on structure and organization, not design.

## Desired Results
- A working webpage introducing myself professionally and personally
- A scope document that helped me think through what the page needed before I started building it
- Practice writing clean and readable HTML without overcomplicating it
- A page that loads correctly in a browser and is straightforward to read

## Exclusions
- No CSS styling...keeping the focus on structure
- No JavaScript or other features
- Single page only
- No images or media

## Feedback Loops
The main feedback for this project comes from Professor Whalen via grading against the rubric. I reviewed the page myself in a browser before submitting to make sure it works correctly.

## People Resources
The main person working on this project is me. 

## Priorities
Quality is priority because the page needs to meet the assignment requirements and be structured correctly. Time is next because the project has to be submitted by the 4/5 deadline. 

## Acceptance Criteria
The project will be complete when my .md and .html files are submitted in zip file, HTML works in a browser, and all parts from the prompt are included.
